THE PROBLEM

Every developer has a graveyard of forgotten commands.

· Googling the same Docker command for the 40th time
· Jumping to browser notes mid-terminal session
· Onboarding teammates with "just look at my .zshrc"
◈ SNIPPET-SNAP FIXES THIS
· Save once. Find in under 100ms with fuzzy search.
· Stay in the terminal. snip find never breaks your flow.
· snip export → teammate runs snip import. Done.

Everything you need,
nothing you don't.

01

Find anything in milliseconds.

An interactive TUI that filters your entire library as you type. No IDs to remember.

$ snip find docker
▸ docker-clean BASH
  docker-ps BASH
  docker-compose YAML
02

Snippets that fill themselves.

Use {{VAR}} placeholders. snip copy prompts for each value before copying — no editing needed.

$ snip copy 3
  HOST › api.example.com
  PORT › 8080
✓ Copied (2 vars resolved)
03

Readable at a glance.

Language-aware highlighting in the preview pane and in snip list. Supports 250+ languages.

# deploy.sh
if [ $ENV = "prod" ]; then
  docker compose up -d
fi
04

macOS. Linux. Windows.

Single binary. No runtime dependencies. Install via Homebrew, Scoop, or direct download.

# macOS / Linux
brew install snippet-snap

# Windows
scoop install snippet-snap

Get started in seconds.

brew install O-Aditya/tap/snippet-snap
curl -sSL https://raw.githubusercontent.com/O-Aditya/snippet-snap/main/scripts/install.sh | bash
scoop bucket add snippet-snap https://github.com/O-Aditya/scoop-bucket
scoop install snippet-snap

Six commands. That's the whole tool.

Command Description Example
snip add Save a new snippet interactively snip add --name curl-auth --lang bash --tags "api,auth"
snip list Print all saved snippets as a flat table snip list --lang python
snip find ★ Launch fuzzy-search TUI snip find
snip copy Copy to clipboard — fills {{VARS}} interactively snip copy 3
snip edit Open snippet in $EDITOR snip edit docker-clean
snip rm Remove a snippet by ID snip rm 5
